Protocol Layer

Five Fixed Event Types

Event types are hardcoded into the protocol itself — not registry-driven. Everything beneath them is extensible.

```
FAULT
Fault

A critical failure condition requiring immediate attention or intervention.

WARNING
Warning

A non-critical condition indicating degraded performance or approaching limits.

MAINTENANCE
Maintenance

Scheduled or triggered maintenance events reported by the machine.

STATE_CHANGE
State Change

Transition between operational modes, active states, or configurations.

RECOVERY
Recovery

Confirmation that a previous fault or warning condition has been resolved.

```
Registry Layer

The Code Format

```
CANONICAL CODE STRUCTURE
DOMAIN.SYSTEM.FAULT.SEVERITY
1
DOMAIN Top-level machine system area (e.g. PROPULSION, NAVIGATION)
2
SYSTEM Subsystem within the domain (e.g. MOTOR, GPS, BATTERY)
3
FAULT TYPE Registry-defined fault class (e.g. OVERHEAT, SIGNAL_LOSS)
4
SEVERITY S0 (diagnostic) through S5 (immediate stop)

All domains, systems, and fault types are defined in the versioned registry — allowing the vocabulary to evolve independently of the protocol itself.

PROPULSION.* Motors, thrusters, fuel systems
NAVIGATION.* GPS, IMU, path planning
POWER.* Batteries, charging, voltage rails
SENSING.* Cameras, LiDAR, radar, proximity
COMMS.* Wireless links, telemetry, protocols
+ more in registry → Browse full registry
```
